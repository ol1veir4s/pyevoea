In this post, we focus on the `max_q` function, which helps you select a single community partition from the Pareto front based on the modularity score \( Q \). By choosing the solution with the maximum \( Q \), the function provides a recommendation that balances intra-community density and inter-community sparsity.

### Step-by-Step Example

1. **Create a NetworkX Graph**

   Begin by creating a graph. We again use the Karate Club graph as an example:

   ```python
   import networkx as nx

   # Create the Karate Club graph
   G = nx.karate_club_graph()
   ```

2. **Initialize the Community Detection Instance**

   Import the detection module and create an instance. Use the same parameters as before:

   ```python
   import pyevoea

   # Initialize the community detection algorithm
   detector = pyevoea.MocdNsgaII(
       graph=G,
       debug_level=1,
       pop_size=100,
       num_gens=500,
       cross_rate=0.8,
       mut_rate=0.2
   )
   ```

3. **Select the Best Partition Using max_q**

   Call the `max_q` method to select the community partition with the highest modularity \( Q \). This function internally computes the modularity for each candidate and returns the best one.

   ```python
   # Get the recommended partition with maximum modularity
   best_partition = detector.max_q()

   # Print the resulting partition
   print("Recommended Partition:", best_partition)
   ```

The `max_q` function operates on the Pareto front generated by the NSGA-II process. It computes the modularity \( Q \) and then selects the partition with the maximum \( Q \). This single partition is chosen as it represents the best trade-off between the objectives, and is expected to reveal the most significant community structure in your network.
